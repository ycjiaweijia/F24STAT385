Regular expression

Regular expressions provide a flexible way to search or match string patterns in text. A single expression, commonly called a regex, is a string formed according to the regular expression language. We’ll use regular expression functions from stringr and tidyr.

For details of the regular expression language in R, please read the stringr cheat sheet. There are also many great websites for learning regex. This website is one example.

We will briefly mentioned a few rules here.

  • .: matches any character except a newline.

  • \d [please type \\d]: matches any digit. It is the same as [0-9].

  • \D [please type \\D]: matches any characters that are NOT \d. It is the same as [^0-9].

  • \w [please type \\w]: matches any alphabetic or numeric character. It is the same as [a-zA-Z0-9_].

  • \W [please type \\W]: matches any characters that are NOT \w.

  • \s [please type \\s]: matches any white spaces. It is the same as [\t\n\r\f\v].

  • \S [please type \\S]: matches any characters that are not \s.

  • \A [please type \\A]: matches the start of the string.

  • \Z [please type \\Z]: matches the end of the string.

  • *: Causes the resulting RE to match 0 or more repetitions of the preceding RE, as many repetitions as are possible.

  • +: Causes the resulting RE to match 1 or more repetitions of the preceding RE, as many repetitions as are possible.

  • ?: Causes the resulting RE to match 0 or 1 repetitions of the preceding RE.

  • *?, +?, ??: The *, +, and ? qualifiers are all greedy; they match as much text as possible. Adding ? after the qualifier makes it perform the match in non-greedy or minimal fashion; as few characters as possible will be matched.

  • {m}: Specifies that exactly m copies of the previous RE should be matched.

  • {m,n}: Causes the resulting RE to match from m to n repetitions of the preceding RE, attempting to match as many repetitions as possible.

  • {m,n}?: Causes the resulting RE to match from m to n repetitions of the preceding RE, attempting to match as few repetitions as possible.

  • []: Used to indicate a set of characters.

  • (): set groups.

Note

To search multiple characters simultaneously, you may use []. For example, [abc] means either a or b or c. However, [] doesn’t recognize special characters, so [\\s|\\w] means either \ or s or \ or w, instead of the pattern \s or \w.

To search such a pattern, you may use (|). For example, (\\s|\\w) means either \s or \w satisfies the pattern.

Question 1.

Please use regular expressions to finish the following tasks. Let vec = c("a", "aBb", "acBbbb", "CaB", "c aB b", "aB\t", "abbBcb", "abc", "abbBb", "aBbb", "abbbBb").

  1. [2 pts] Match a string that contains an a followed by zero or more b or B’s. Please first define the regular expression as rx1a. Then use vec_1a <- str_detect(vec, rx1a) to save the result. Please do the similar for part (b) – (d).
vec = c("a", "aBb", "acBbbb", "CaB", "c aB b", "aB\t", "abbBcb", "abc", "abbBb", "aBbb", "abbbBb")
rx1a <- "a[bB]*"
str_view(vec, rx1a)
#>  [1] │ <a>
#>  [2] │ <aBb>
#>  [3] │ <a>cBbbb
#>  [4] │ C<aB>
#>  [5] │ c <aB> b
#>  [6] │ <aB>{\t}
#>  [7] │ <abbB>cb
#>  [8] │ <ab>c
#>  [9] │ <abbBb>
#> [10] │ <aBbb>
#> [11] │ <abbbBb>
vec_1a <- str_detect(vec, rx1a)

# (a)' If match a string that consists of an `a` followed by zero or more `b` or `B` characters and contains nothing else.
rx1a1 <- "\\ba[bB]*\\b"
str_view(vec, rx1a1)
#>  [1] │ <a>
#>  [2] │ <aBb>
#>  [5] │ c <aB> b
#>  [6] │ <aB>{\t}
#>  [9] │ <abbBb>
#> [10] │ <aBbb>
#> [11] │ <abbbBb>
str_detect(vec, rx1a1)
#>  [1]  TRUE  TRUE FALSE FALSE  TRUE  TRUE FALSE FALSE  TRUE  TRUE  TRUE
  1. [2 pts] Match a string that contains an a followed by one or more bor B’s.

rx1b <- "a[bB]+"
vec_1b <- str_detect(vec, rx1b)
  1. [2 pts] Match a string that contains an a followed by zero or one b or B’s.
rx1c <- "a[bB]?"
vec_1c <- str_detect(vec, rx1c)
  1. [2 pts] Match a string that contains an a followed by three b or B characters, or a mixture of both.
rx1d <- "a[bB][bB][bB]"
rx1d <- "a[bB]{3}"

vec_1d <- str_detect(vec, rx1d)

Question 2.

[2 pts] Find all words starting with a or e in the given string below:

text = "The following example creates an ArrayList with a capacity of 50 elements. Four elements are then added to the ArrayList and the ArrayList is trimmed accordingly."

Please first define the regular expression as rx2. Then use text_selected <- str_extract_all(text, rx2) to save the result.


rx2 <- "\\b[ae]\\w*"
text_selected <- str_extract_all(text, rx2)
text_selected

#> [1] "example"     "an"          "a"           "elements"    "elements"

Question 3.

[2 pts] From ser = c('Apple', 'Orange', 'Plan', 'Python', 'Money'), we need to find the words that contain at least 2 vowels. Please first define the regular expression as rx3. Then use ser_selected <- str_detect(ser, rx3) to save the result.

ser = c('Apple', 'Orange', 'Plan', 'Python', 'Money')

rx3 <- "\\w*[aeiouAEIOU]\\w*[aeiouAEIOU]\\w*"

ser_selected <- str_detect(ser, rx3)
ser_selected
#> [1]  TRUE  TRUE FALSE FALSE  TRUE
